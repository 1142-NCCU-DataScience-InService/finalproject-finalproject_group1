# ⚖️ 車禍案件：精神慰撫金 AI 預測系統 (Predicting Mental Damages in Traffic Accidents)

本專案為資料科學期末專題，旨在運用機器學習技術，分析司法院裁判書開放資料，建立「車禍案件精神慰撫金」的預測模型。透過此模型，協助實務工作者與一般民眾預估合理的賠償金額，減少訴訟資源浪費。

## 🎯 1. 專案目標與輸入資料 (Input)

### 1.1 研究目標 (Goal)
- **核心問題**：精神慰撫金在實務上往往由法官依「自由心證」裁量，缺乏明確公式。
- **解決方案**：透過大數據分析過往判決，找出影響判決金額的關鍵特徵，並建立預測模型。

### 1.2 資料來源 (Data Source)
- **來源**：司法院裁判書開放資料 (JSON 格式)
- **範圍**：2023 年 01 月至 2025 年 04 月 (民國 112-114 年)
- **樣本數**：擷取「損害賠償」且包含「車禍/交通事故」之有效案件共 **7,536 筆**。

### 1.3 資料預處理 (Preprocessing)
1. **處理缺失值 (Missing Data)**：醫療費、看護費、工作損失若未記載則補 0（視為未請求）。剔除缺乏目標變數（慰撫金）的無效資料。
2. **清理異常值 (Outliers)**：剔除極端判決（設定慰撫金界線為 1萬至 1000萬），最終保留 **7,050 筆**高品質資料。
3. **特徵縮放 (Scale value)**：由於金額特徵絕對值極大（動輒數百萬），對所有金額特徵進行 **Log Transform (對數轉換)**，以穩定模型訓練。
4. **特徵編碼**：對傷亡類型進行順序編碼 (死亡=3, 重傷=2, 傷害=1)，並對審理法院進行 One-Hot Encoding。

---

## 🧠 2. 模型建構 (Modeling)

### 2.1 演算法選擇
針對具有高度非線性關係的表格型特徵，我們選擇使用樹狀模型：
- **Random Forest Regressor (隨機森林迴歸)**

### 2.2 訓練策略
- **Train/Test Split**：依 80% (5,640筆) 與 20% (1,410筆) 隨機切割訓練與測試集。
- **Hyperparameter Tuning**：使用 GridSearchCV 尋找最佳深度 (`max_depth=10`) 與葉節點最小樣本數 (`min_samples_leaf=5`) 以防止 Overfitting。

### 2.3 基準模型 (Null Model)
為了評估機器學習模型的真實成效，我們建立了一個 Null Model：
- **策略**：無論案件特徵為何，一律「盲猜」訓練集的精神慰撫金平均值。
- **基準誤差**：MAE 高達 494,720 元。

---

## 📊 3. 成效評估 (Results)

### 3.1 評估指標 (Metrics)
我們選用 **MAE (平均絕對誤差)** 作為主要指標，因為比起 RMSE，MAE 更能直觀反映「預測金額與法官真實判決金額，平均差了幾元」，在法律實務上最具解釋力。

### 3.2 顯著進步 (Significant Improvement)
- **Null Model MAE**: 494,720 元
- **Random Forest MAE**: 304,969 元
- **進步幅度**: 我們的模型比盲猜基準**顯著提升了 38.4% 的準確度**！R-squared 達到 0.4618。

### 3.3 特徵重要性 (Feature Importance)
影響法官判決精神慰撫金的 Top 3 關鍵因素：
1. `Care_Fee` (看護費用) - 39%
2. `Injury_Level` (傷亡嚴重度) - 28%
3. `Medical_Fee` (醫療費用) - 18%
> **洞察**：實務上，被害人需要的照護費用與醫療支出越高，法官越容易認定其受到的精神痛苦巨大，進而判賠更高的慰撫金。

## 💻 4. 系統展示與重現 (Demo & Reproducibility)

### 4.1 線上視覺化 (On-line visualization)
我們使用 `Streamlit` 開發了互動式的 Web App。使用者可輸入案件條件（如傷亡程度、醫療費、是否酒駕等），系統將即時呼叫訓練好的模型並試算賠償金額。

### 4.2 如何重現專案 (How to reproduce)
1. 套件安裝：`pip install -r requirements.txt`
2. 資料清洗：執行 `python 03_exploratory_analysis.py` 產出 `dataset_cleaned.csv`
3. 模型訓練：執行 `python 04_model_training.py` 訓練模型並產出 `rf_model.pkl`
4. 啟動展示：執行 `streamlit run 05_demo_app.py` 即可開啟網頁。

### 4.3 專案挑戰 (Challenges)
- **非結構化資料擷取**：司法文書格式極不統一。最大的挑戰在於使用 Regular Expression (正則表達式) 從數十萬字的中文裁判書中，精準提取各項賠償金額並排除和解、調解案件。
